## 产品核心定位（2026-05-28 更新）

**我们在做的不是照片整理工具，是个人神话生成引擎。**

> 日常之物，冠上非常之感。

用户的相册里不需要有精美的照片。街头的一朵花、桌上的纸巾、地上的雨伞——这些散漫的日常物件，通过文学镜头的折射，变成有叙事张力的东西。这是我们和所有大厂工具的根本区别。

**VLM 幻觉哲学：**
大厂把 VLM 的"幻觉"当 bug 修。我们把它当特性用。模型把白狗看成鹿、把飞机窗外看成"水面"——这是它在对图像做神话化解读。这种失真和山海经的作者"描述从未见过的三头兽"在本质上是一样的事情。我们的叙事引擎拥抱这种失真，并赋予它意义。

**最终平台形态：**
不只是一个 app，而是一个**可 fork 的叙事引擎 + 文学插件商店**。每个"文学镜头"（prompt 模板 + UI 风格 + 聚类逻辑）是一个可独立分发的插件。我们做好引擎和接口，欢迎有创造力的人接入自己的文学框架。

### 视觉 AI 选型 — 2026-05-28 四模型 Benchmark 结论

**选型已确定，两层架构：**

```
第一层（全库批量标注）  qwen2.5vl:7b   41s/张  一致性优先，无 API 依赖
  → 全部 46K 张，结构化字段（has_people, mood, setting 等）
  → narrative_hint 作为语义索引，不追求文学质量

第二层（精选精标注）   mimo-v2.5（有网）或 gemma4:e4b（离线）
  → 聚类后的代表性照片，每个叙事模式的"主角照片"
  → 追求文学质量，拥抱失真和幻觉，这是叙事变形的材料
```

**四模型测评结果（30张固定集）：**

| 模型 | 均速 | 叙事质量 | 幻觉抵抗 | 适用 |

| mimo-v2.5 | 17.4s（云端）| ⭐⭐⭐⭐⭐ | 最稳 | 精标注主力 |
| qwen2.5vl:7b | 41.3s | ⭐⭐⭐⭐ | 良好 | 批量标注主力 |
| gemma4:e4b | 60.7s | ⭐⭐⭐⭐⭐ | 良好 | 精标注离线备选 |
| qwen2.5vl:3b | 12.9s | ⭐⭐⭐ | 有幻觉 | 快速预览 |

## NEXT · 六月下旬（2026-06-15 ~ 06-30）

### 每天 20 张日常叙事流（产品核心节奏）

> 从昨晚讨论提炼：每日 20 张是让用户持续使用的关键机制。

**设计原则：**
- 不是用户选，是算法根据叙事需要选（今天《酒吧长谈》需要一张 2019 年傍晚，就去取它）
- 20 张，5 分钟，一个小章节
- 每天结束后有小小的「今日叙事」归档感

**实现路径：**
- [ ] 后端：`GET /api/daily/narrative`，基于日期 seed + 叙事模式返回 20 张
- [ ] 前端：新的日常入口，从微光之门或单独入口进入
- [ ] 需要讨论：是放在「发现」世界里，还是单独作为一个入口？

### 《酒吧长谈》时间回响算法

> 核心洞察：不只是「同一时刻跨年份」，而是现在的你在读过去的你——记忆再巩固。

- [ ] 后端：`novel_cathedral.py`
  - 同小时（±30min）跨年份聚合
  - 加权：年份差越大权重越高（更戏剧化的「对话」）
  - 可选维度：同季节、同 camera_model
- [ ] 前端：时间轴布局，左右对话感，不是列表
- [ ] UI 哲学：两张照片之间的「解释距离」——2019 vs 2024，你现在还记得吗？

### 100 张 Onboarding 流程（面向未来用户）

- [ ] 进入 APP 时可选：「先带 100 张体验」vs「导入完整图库」
- [ ] 快速扫描 + 立即进入叙事引擎（< 2 分钟）
- [ ] 体验完再问：「你还有更多记忆吗？」
- [ ] 需要讨论：默认 100 张还是 20 张？从哪个文件夹选？

## LATER · 七月以后

### 《看不见的城市》视觉 AI 城市

- 依赖：MiniCPM-V 本地部署成功 + 感知需求矩阵确认
- VLM 对每张照片提取主题标签 → 聚类 → 命名「城市」（AI 命名 or 用户命名）
- 城市里的照片跨越年份 —— 你一直在拍「窗」「路口」「夜晚餐厅」

### 《山海经》GPS 神话地理

- 依赖：GPS 反地理编码补全（目前 ~0% 覆盖）
- 将走过的地方构成个人地理志，不是地图，是诗
- 依赖外置硬盘上的原始 EXIF GPS 数据

### APP 打包与分发

> 用户问题四：不要让用户自己配环境。

**CLI 一键安装（中期目标）：**
```bash
curl -sSL unearth.app/install.sh | sh
```
自动处理：Python 环境、SQLite 初始化、MiniCPM-V 下载（~2-3GB）、前端 build、首次启动。

**桌面 APP（长期目标）：**
- Tauri 打包（比 Electron 轻，Rust 内核）
- 模型作为首次启动后台下载
- 面向不懂命令行的普通用户

### 叙事引擎 + 插件商店（平台化）

> **2026-05-28 升级：** 这不再是"未来某天"的方向，而是整个产品的终极形态。

**核心理念：** 我们只做引擎和接口。每个"文学镜头"是一个可独立 fork / 分发的插件：
- prompt 模板（VLM 感知配置）
- UI 风格（视觉语言）
- 聚类逻辑（照片如何被组织）
- 叙事参照（对应哪部作品或文化系统）

**插件 = 一套配置文件，任何人可以 fork。**

实现路径：
1. 先把我们自己的三个模式（日常生活 / 山海经 / 酒吧长谈）做成标准插件格式
2. 将插件规范文档化：感知需求 → VLM prompt → 聚类算法 → UI 骨架
3. 开放接口，接受社区贡献（例：《百年孤独》魔幻现实主义镜头、《物语》系列、《请回答1988》怀旧镜头）
4. 哈扎尔词典界面作为插件商店的主入口
