# 显影 Unearth · 设计语言规范
### DESIGN_LANGUAGE.md
> 所有 UI 改动必须先读此文件。这是视觉和交互的唯一标准。
> 优先级高于任何之前写的视觉描述。

---

## 核心意象

> 你走进一个岩洞。
> 洞壁上嵌着发光的东西。
> 你伸手，取出一件，看清它是什么。
> 然后决定：带走，还是让它留在这里。

所有界面决策都从这个意象出发。
**不是文件管理器。不是照片查看器。是一次进入记忆深处的行动。**

## 一、字体系统

### 字体选择
```
主字体（中英文）：系统字体回退链
  "DM Sans", "PingFang SC", "Noto Sans SC", sans-serif

特殊标题（大标题、仪式感文字）：
  "Cormorant Garamond", "Noto Serif SC", serif
  → 用于「这一组挖完了」「第一个瞬间从地层里浮现」等仪式性文字
```

### 字号层级
```
--text-display   : 32px / line-height 1.2 / weight 500
  用于：「这一组挖完了」「显影」主标题

--text-title     : 22px / line-height 1.3 / weight 500
  用于：年份「2023」，月份「11月」，事件标题

--text-nav       : 16px / line-height 1.4 / weight 400
  用于：面包屑导航，按钮文字

--text-body      : 15px / line-height 1.6 / weight 400
  用于：事件描述，统计数字

--text-meta      : 13px / line-height 1.5 / weight 400
  用于：上下文卡片（日期/地点/今日张数）
  颜色不低于 rgba(255,255,255,0.75)，确保可读

--text-caption   : 11px / line-height 1.4 / weight 300
  用于：快捷键提示，极次要信息
```

### 禁止事项
- 禁止正文使用 weight 200/300（太细，不可读）
- 禁止重要信息字号低于 13px
- 禁止纯白文字直接放在照片上（需要有背景层）

## 三、材质与质感

### 岩层纹理
所有深色背景区域加细腻噪点纹理，不明显但有质感：
```css
.strata-texture {
  background-image: url("data:image/svg+xml,..."); /* SVG噪点 */
  /* 或使用 CSS filter: contrast + brightness 叠加 */
}
```

### 玻璃态面板（统一规范）
```css
.glass-panel {
  background: var(--glass-bg);
  backdrop-filter: blur(var(--glass-blur));
  border: 1px solid var(--glass-border);
  border-radius: 12px;
}
```

### 发光效果
已完成/激活状态使用 box-shadow 发光，不用 border：
```css
/* 激活状态 */
box-shadow: 0 0 20px rgba(142, 173, 196, 0.3);
/* 书候选 */
box-shadow: 0 0 16px rgba(232, 200, 120, 0.4);
```

## 四、空间语言

### 地层视图（StrataView）— 重设计方向

**现在的问题：** 横铺小格子，局促，看不清年份。

**应该是：** 真正的地质切面——宽阔的横条，有厚度，有重量。

```
布局规则：
- 每一年是一个完整的横条，高度最小 120px
- 横条内，月份是不等宽的分段（宽度∝照片数量）
- 年份数字大而清晰：--text-title，左侧固定显示
- 横条之间有明显间距（24px），强调层级感
- 已完成的月份：发光 + 清晰
- 未处理的月份：半透明磨砂，像是还埋在岩层里

交互：
- hover 某个月份：该月份轻微上浮（translateY -4px），显示详细信息
- click：平滑缩放进入（不是跳转，是「走进去」）
```

### 事件列表（SiteView）— 重设计方向

**现在的问题：** 一行一行的列表，Lightroom Classic 感。

**应该是：** 考古现场的发掘坑俯视图——不规则的区块，大小各异。

```
布局规则：
- Masonry（瀑布流）或不规则网格，不是等高的列表行
- 每个事件是一张「发掘坑卡片」：
    封面照片铺满背景（模糊 + 压暗）
    叠加：事件时间、张数、地点（如有）
    底部：进度条（已决策/总数）
- 卡片大小和照片数量相关（多的卡片更大）
- 未开始：半透明，像是还在土里
- 进行中：清晰，边框微光
- 已完成：发光效果，缩略polaroid角标

交互：
- hover：卡片内封面图轻微缩放（scale 1.03），营造「走近」感
- click：黑幕过渡（预告幕），不是直接跳转
```

### 决策界面（DecisionView）— 核心交互重设计

**现在的问题：** 照片平铺在屏幕上，横向缩略图条，三个按钮——Lightroom。

**应该是：** 从岩层里取出宝石。

```
主照片区域：
- 照片不占满全屏，四周留有「岩洞空间」感（边距至少 80px）
- 照片有轻微的景深阴影（box-shadow 模拟漂浮感）
- 背景：当前照片高斯模糊铺满，亮度压至 15-20%，加噪点纹理

缩略图条 → 改为「岩层序列」：
- 不是横向平铺的小图
- 而是右侧或底部的「队列」，像是一叠从岩层里待取出的东西
- 当前照片：大，居中，「在手中」
- 待决策照片：右侧半透明叠放，暗，像还在岩层里
- 已决策照片：消失（带走浮上去，留下沉下去）

决策动画（关键）：
  带走（K）：
    照片向上浮起 → scale(1.05) translateY(-20px)
    发出冷绿色光晕
    然后缩小收进右上角「记忆囊」
    下一张从「岩层队列」浮现到中央

  留在这里（D）：
    照片向下沉 → scale(0.95) translateY(20px) opacity(0)
    背景石纹短暂「吸收」它（涟漪动画）
    下一张浮现

  浮现（每张新照片出场）：
    从 brightness(0) blur(8px) scale(0.95)
    → brightness(1) blur(0) scale(1)
    duration: 0.4s，感觉像从黑暗里显现

操作区域：
- 三个按钮改为更有仪式感的设计
- 「留在这里」和「带走」应该是主要的两个，视觉上更大
- 「稍后」是次要的，opacity 降低，不争夺注意力
- 快捷键提示：kbd 键帽样式，小而克制
```

## 五、动画原则

### 缓动曲线
```css
--ease-out-expo : cubic-bezier(0.16, 1, 0.3, 1)   /* 主要动作：快进慢出 */
--ease-in-out   : cubic-bezier(0.4, 0, 0.2, 1)     /* 状态切换 */
--ease-spring   : cubic-bezier(0.34, 1.56, 0.64, 1) /* 弹性：轻微过冲 */
```

### 时长规范
```
即时反馈（按钮按下）  : 100-150ms
照片切换过渡          : 350-450ms
界面层级切换          : 500-600ms
仪式性动画（里程碑）  : 800ms+
```

### 禁止事项
- 禁止无意义的 bounce 动画（廉价感）
- 禁止超过 600ms 的非仪式性过渡（卡顿感）
- 禁止同时有超过两个动画在运行（混乱）

## 六、实现优先级

按体验影响从高到低：

```
第一优先（影响整体可读性）：
  → 字体系统升级（字号 + 字重 + 行高）
  → 上下文卡片可读性（颜色 + 字号）

第二优先（影响核心体验）：
  → 决策界面：照片浮现动画升级
  → 决策界面：「岩层队列」替代横向缩略图条

第三优先（影响进入体验）：
  → StrataView 地层视图重设计（横条 + 厚度）
  → SiteView 事件列表重设计（Masonry卡片）

第四优先（完善细节）：
  → 按钮物理感精调
  → 背景噪点纹理
  → 所有界面的间距规范统一
```

## 给 Claude Code 的工作规则

1. **每次改动前先读这个文件**
2. **改 UI 时，问自己：这符合「走进岩洞」的意象吗？**
3. **不要为了实现方便而妥协视觉方向**
4. **字号不得小于规范值，颜色不得低于规范对比度**
5. **每完成一个优先级的改动，截图确认，再进行下一个**
